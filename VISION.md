# RepoMind 项目愿景

## 终极目标
打造 macOS 原生代码仓库 AI 知识管家，让开发者在任何代码库中都像工作了三年的老员工。

## 核心价值主张
1. **GitHub 云端直连** — 一键连接所有仓库，无需手动 Clone
2. **实时增量索引** — Git Webhook 自动更新知识图谱，永不过期
3. **原生 Mac 体验** — DMG 一键安装，菜单栏驻留，Spotlight 快速搜索
4. **双路智能搜索** — BM25 全文 + CodeBERT 语义向量融合
5. **AI 深度集成** — Claude ​Code / Cursor MCP 工具，自动架构分析

## 技术约束与原则

### 架构原则
- **离线优先** — 所有核心功能在没网络时也能用
- **本地优先** — 数据默认不上云，隐私优先
- **轻量级** — Tauri < Electron，无不必要的依赖
- **可扩展** — 模块化设计，便于插件化

### 性能标准
- 菜单栏搜索：< 200ms 响应
- 仓库索引（1k 文件）：< 2 分钟
- 增量更新：< 10 秒
- 图谱查询：< 500ms

### 代码质量
- TypeScript 100% 覆盖
- 单元测试 > 80% 覆盖率
- Rust 无 unsafe 块（除非特殊性能理由）
- 零 hardcoded secrets

## 风险与应对

| 风险 | 概率 | 影响 | 应对 |
|------|------|------|------|
| Tree-sitter 类型解析不准 | 高 | 中 | LSP 补充 + 用户手动验证置信度 |
| GitHub API rate limit | 中 | 中 | 本地缓存 + 增量同步策略 |
| Tauri 生态不成熟 | 低 | 高 | Fallback Electron（备选） |
| 大型仓库（100k+ 文件）索引时间 | 中 | 中 | 渐进式索引 + 后台处理 |
| 云端同步用户少 | 中 | 低 | MVP 先做本地版本 |
